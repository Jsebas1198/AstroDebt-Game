¡ATENCIÓN! Docker + Pygame NO funciona bien en Windows porque:
  ❌ Pygame necesita una ventana gráfica (GUI)
  ❌ Docker en Windows no tiene acceso a la pantalla
  ❌ docker-compose up -d NO mostrará el juego

🐳 ¿Para qué sirve Docker entonces?
════════════════════════════════════════════════════════════════════════════

Docker es útil para:
  ✅ Ejecutar tests sin GUI
  ✅ Verificar que las dependencias están bien
  ✅ CI/CD (integración continua)
  ✅ Desarrollo sin interfaz gráfica

Comandos útiles de Docker:
──────────────────────────
  # Shell interactivo para trabajar sin GUI
  docker-compose run dev bash

  # Ejecutar tests
  docker-compose run dev pytest

  # Verificar imports
  docker-compose run dev python -c "import pygame; print('OK')"

  # Ejecutar solo tests (servicio dedicado)
  docker-compose run test
