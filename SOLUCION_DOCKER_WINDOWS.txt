  ✅ SÍ PUEDES USAR DOCKER SIN INSTALAR PYTHON EN WINDOWS
════════════════════════════════════════════════════════════════════════════

¡ENTENDIDO! Quieres usar Docker para tener Python sin instalarlo en tu máquina.
Eso es perfectamente posible, solo necesitas UN paso adicional.

════════════════════════════════════════════════════════════════════════════

🔧 SOLUCIÓN: Instalar VcXsrv (Servidor de Ventanas)
════════════════════════════════════════════════════════════════════════════

VcXsrv es un programa pequeño que permite que Docker muestre ventanas en Windows.
Es como un "traductor" entre Docker y tu pantalla de Windows.

────────────────────────────────────────────────────────────────────────────
  PASO 1: Descargar VcXsrv (SOLO UNA VEZ)
────────────────────────────────────────────────────────────────────────────

  Descarga de: https://sourceforge.net/projects/vcxsrv/
  
  → Descarga el archivo .exe
  → Instálalo (Next, Next, Finish)
  → ¡Eso es todo para este paso!

────────────────────────────────────────────────────────────────────────────
  PASO 2: Configurar VcXsrv (CADA VEZ QUE VAYAS A USAR DOCKER)
────────────────────────────────────────────────────────────────────────────

  1. Busca "XLaunch" en el menú de Windows
  
  2. Configuración:
     Pantalla 1: ● Multiple windows → Next
     Pantalla 2: ● Start no client → Next
     Pantalla 3: ☑ Disable access control ← ¡IMPORTANTE!
                 ☑ Native opengl
                 → Next → Finish
  
  3. Verás un icono "X" en la bandeja del sistema

────────────────────────────────────────────────────────────────────────────
  PASO 3: Ejecutar Docker con el Script Automático
────────────────────────────────────────────────────────────────────────────

  ¡Ahora es SUPER FÁCIL! Solo ejecuta:
  
    iniciar-docker-windows.bat
  
  Este script hace TODO automáticamente:
    ✓ Verifica Docker Desktop
    ✓ Verifica VcXsrv
    ✓ Detecta tu IP automáticamente
    ✓ Configura Docker Compose
    ✓ Construye la imagen (primera vez)
    ✓ Ejecuta el juego
  
  ¡Y listo! La ventana del juego debería aparecer.

════════════════════════════════════════════════════════════════════════════

📋 RESUMEN DE LO QUE NECESITAS
════════════════════════════════════════════════════════════════════════════

  Software a instalar (SOLO UNA VEZ):
    □ Docker Desktop (ya lo tienes ✓)
    □ VcXsrv (descargar e instalar)
  
  Python en tu máquina:
    □ NO es necesario ✓
  
  Cada vez que quieras trabajar:
    1. Inicia Docker Desktop
    2. Inicia VcXsrv (XLaunch)
    3. Ejecuta: iniciar-docker-windows.bat

════════════════════════════════════════════════════════════════════════════

🚀 INICIO RÁPIDO (Después de instalar VcXsrv)
════════════════════════════════════════════════════════════════════════════

  1. Inicia Docker Desktop
  
  2. Inicia XLaunch (busca en menú de Windows)
     - Multiple windows
     - Start no client  
     - ☑ Disable access control
     - ☑ Native opengl
  
  3. Ejecuta:
     
     cd "C:\Projects\Hackaton Python\nave_varada"
     iniciar-docker-windows.bat

  ¡La ventana del juego debería abrirse!

════════════════════════════════════════════════════════════════════════════

🛠️ ALTERNATIVA: Configuración Manual
════════════════════════════════════════════════════════════════════════════

Si prefieres configurar manualmente:

  1. Obtén tu IP de Windows:
     
     ipconfig
     
     Busca "Dirección IPv4" (ejemplo: 192.168.1.100)
  
  2. Edita: docker-compose.WINDOWS-GUI.yml
     
     Cambia "TU_IP_AQUI" por tu IP real
  
  3. Ejecuta:
     
     docker-compose -f docker-compose.WINDOWS-GUI.yml build
     docker-compose -f docker-compose.WINDOWS-GUI.yml up astrodebt

❓ PREGUNTAS FRECUENTES
════════════════════════════════════════════════════════════════════════════

P: ¿Por qué necesito VcXsrv?
R: Docker en Windows no puede crear ventanas directamente. VcXsrv es el "puente"
   que permite que las aplicaciones en Docker muestren ventanas en Windows.

P: ¿Es complicado?
R: ¡No! Instalas VcXsrv una vez, y luego solo ejecutas iniciar-docker-windows.bat

P: ¿Tengo que instalar Python en mi máquina?
R: ¡NO! Python está dentro de Docker. Tu máquina Windows no necesita Python.

P: ¿Esto es seguro?
R: Sí, VcXsrv solo se usa localmente en tu máquina. Ciérralo cuando no lo uses.

P: ¿Hay otra forma más simple?
R: Sí, instalando Python localmente con run.bat, pero entiendo que prefieres
   no instalar Python en tu máquina.

🔧 SOLUCIÓN DE PROBLEMAS
════════════════════════════════════════════════════════════════════════════

Error: "Cannot connect to display"
  → Verifica que VcXsrv está corriendo (icono X en bandeja)
  → Verifica que marcaste "Disable access control"
  → Reinicia VcXsrv

Error: "Connection refused"
  → Permite VcXsrv en el Firewall de Windows
  → Ve a: Panel de Control → Firewall → Permitir app
  → Busca y permite vcxsrv.exe

La ventana no aparece
  → Verifica que Docker Desktop está corriendo
  → Verifica los logs: docker-compose logs astrodebt
  → Reinicia VcXsrv y vuelve a intentar
